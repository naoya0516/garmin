# 03. 画面設計

## 画面一覧

MVPでは以下の1画面のみ。

| 画面ID | 画面名 | 概要 |
|---|---|---|
| S-01 | アクティビティ一覧画面 | F-01（同期）・F-02（一覧表示）を提供するメイン画面 |

## レイアウト（S-01 アクティビティ一覧画面）

```
┌──────────────────────────────────────────────┐
│ Garmin Activity Tracker          [同期ボタン]  │
├──────────────────────────────────────────────┤
│ 日付      │種別   │距離   │時間   │ペース │平均心拍│カロリー│
├──────────────────────────────────────────────┤
│ 2026-07-16│running│5.2km │28:30 │5:29/km│152bpm │320kcal │
│ 2026-07-14│walking│3.0km │35:10 │11:43/km│105bpm │180kcal │
│ ...       │       │      │      │       │       │        │
└──────────────────────────────────────────────┘
```

- ヘッダー: アプリ名 + 同期ボタン（[SyncButton.tsx](../frontend/src/components/SyncButton.tsx) 相当）
- テーブル本体: [ActivityTable.tsx](../frontend/src/components/ActivityTable.tsx) 相当。列は日付・種別・距離・時間・ペース・平均心拍・カロリー
- 同期ボタン押下中はローディング表示、完了後にテーブルが自動更新される
- 一覧が0件の場合は「アクティビティがありません。同期してください」等の空状態メッセージを表示する

## 画面遷移図

MVPは1画面のみのため遷移はないが、将来のカレンダー画面・詳細画面への拡張ポイントを点線で示す。

```mermaid
graph LR
    S01["S-01 アクティビティ一覧画面<br/>(MVP)"]
    S02["カレンダー画面<br/>(将来検討)"]
    S03["アクティビティ詳細画面<br/>(将来検討)"]

    S01 -.->|将来: 日付クリック| S02
    S01 -.->|将来: 行クリック| S03
```

## 操作フロー

### 画面表示時（初回ロード）

```mermaid
sequenceDiagram
    participant B as ブラウザ(S-01)
    participant API as FastAPI
    participant DB as SQLite

    B->>API: GET /api/activities
    API->>DB: SELECT * FROM activities ORDER BY start_time_local DESC
    DB-->>API: アクティビティ一覧
    API-->>B: 200 OK (JSON配列)
    B->>B: テーブルに描画
```

### 同期ボタン押下時

```mermaid
sequenceDiagram
    participant B as ブラウザ(S-01)
    participant API as FastAPI
    participant G as Garmin Connect
    participant DB as SQLite

    B->>API: POST /api/sync
    API->>G: get_activities_by_date(直近30日, 全種別)
    G-->>API: アクティビティ一覧(JSON)
    API->>DB: upsert (activity_idで重複排除)
    DB-->>API: 完了
    API-->>B: 200 OK {fetched, upserted}
    B->>API: GET /api/activities (再取得)
    API-->>B: 200 OK (JSON配列)
    B->>B: テーブルを再描画
```
